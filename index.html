raw.githack.com

The Problem

You've got a raw.githubusercontent.com URL that points to some work-in-progress HTML, JavaScript, or CSS. You'd like to share it with someone for a quick demo, or maybe you'd like to use it in a jsPerf test.

Too bad! Most things served from raw.githubusercontent.com have a text/plain content type and are served with an X-Content-Type-Options: nosniff header, which means HTML won't render, and some browsers won't execute JavaScript or apply CSS.

The Solution

Change the domain! Just replace raw.githubusercontent.com with raw.githack.com and skadoosh, you're loading raw files from your GitHub repo and they're served with appropriate content types. Render HTML, apply CSS, execute JavaScript.

Before

https://raw.githubusercontent.com/user/repo/master/filename.js

After

https://raw.githack.com/user/repo/master/filename.js

It works with gists too! Just replace gist.githubusercontent.com with gist.githack.com in the raw gist URL.

NEW! Now it works even with bitbucket! You simply need to replace bitbucket.org with bb.githack.com.

The Caveat

Use this for testing and for sharing demo code with friends, but don't use it for anything production-related. It's slow, it might break from time to time, and it's intended as a convenience for development, not free static hosting.

The Source

The FAQ

Can I use raw.githack.com on a production website?

No. Don't do this. Not even a little bit. It will make me very sad.

raw.githack.com is a free service to make developers' lives easier when developing and testing and debugging things. It's not free hosting. Use GitHub Pages for that.

What will happen if I hotlink to a raw.githack.com URL on a production website?

Bad things. If you hotlink to a raw.githack.com URL on a site that gets lots of traffic, requests will be throttled. If this heavy traffic continues, requests originating from your site will eventually be dropped.

But I really really want to hotlink to a raw.githack.com URL in production!

We can't always get what we want. You should use GitHub Pages, which is designed for that. Also that's not a question.
